# 产品数量统计方法

## 方法1：使用统计脚本（推荐）

### 运行统计脚本

在命令行中运行：

```bash
npm run stats
```

或者直接运行：

```bash
node 统计产品数量.js
```

### 输出示例

```
========== 产品统计结果 ==========

产品ID | 产品名称      | 选择次数
-----------------------------------
1      | 玩偶          | 5       
2      | 蓝色单肩包    | 3       
3      | 白色单肩包    | 2       
4      | 镜子          | 3       
5      | 石膏香片      | 2       
6      | 充电宝        | 1       
-----------------------------------
总计: 6 个产品，共 16 次选择

========== 按受欢迎程度排序 ==========

1. 玩偶 (ID: 1) - 5 次
2. 蓝色单肩包 (ID: 2) - 3 次
3. 镜子 (ID: 4) - 3 次
4. 白色单肩包 (ID: 3) - 2 次
5. 石膏香片 (ID: 5) - 2 次
6. 充电宝 (ID: 6) - 1 次

统计结果已保存到: data/statistics.json
```

统计结果会自动保存到 `data/statistics.json` 文件中。

## 方法2：使用统计页面（可视化）

### 启动后端服务器

```bash
npm start
```

### 打开统计页面

在浏览器中打开：`http://localhost:3000/统计页面.html`

或者直接双击 `统计页面.html` 文件（需要后端服务器运行）

### 功能特点

- 📊 可视化统计卡片
- 📈 按受欢迎程度排序
- 🔄 一键刷新统计数据
- 📱 响应式设计，支持移动端

## 方法3：使用 API 接口

### 获取统计信息

```bash
curl http://localhost:3000/api/statistics
```

或者在浏览器中访问：`http://localhost:3000/api/statistics`

### 响应示例

```json
{
  "success": true,
  "totalProducts": 6,
  "statistics": [
    {
      "productId": 1,
      "productName": "玩偶",
      "count": 5,
      "submissions": [...]
    },
    ...
  ]
}
```

## 方法4：手动统计（文件浏览器）

在文件浏览器中：

1. 打开 `data/` 或 `data/products/` 目录
2. 按文件名排序
3. 统计每个产品ID开头的文件数量

例如：
- `1_玩偶_*.json` - 统计所有以 `1_玩偶_` 开头的文件
- `2_蓝色单肩包_*.json` - 统计所有以 `2_蓝色单肩包_` 开头的文件

## 推荐使用

- **快速查看**：使用方法1（统计脚本）
- **可视化展示**：使用方法2（统计页面）
- **程序调用**：使用方法3（API接口）
