# 后端运行说明

## 问题说明

如果看到 "Project has no JDK configured" 错误，这是因为：
- 这是一个 **Node.js 项目**，不需要 JDK（Java Development Kit）
- IDE 可能误识别了项目类型，或者您点击了 Java 相关的功能

## 正确的运行方式

### 方法1：使用命令行（推荐）

1. **打开命令行/终端**
   - Windows: 按 `Win + R`，输入 `cmd` 或 `powershell`
   - Mac/Linux: 打开 Terminal

2. **进入项目目录**
   ```bash
   cd C:\Users\30543\Desktop\Questionnaire
   ```

3. **安装依赖**（如果还没安装）
   ```bash
   npm install
   ```

4. **启动服务器**
   ```bash
   npm start
   ```
   或者使用开发模式（自动重启）：
   ```bash
   npm run dev
   ```

5. **看到以下信息表示成功**
   ```
   服务器运行在 http://localhost:3000
   数据保存目录: C:\Users\30543\Desktop\Questionnaire\data
   产品分类目录: C:\Users\30543\Desktop\Questionnaire\data\products
   ```

### 方法2：在 IDE 中运行（VS Code）

1. **打开 VS Code**
2. **打开项目文件夹**
3. **打开终端**（按 `` Ctrl + ` `` 或 `Terminal > New Terminal`）
4. **运行命令**
   ```bash
   npm start
   ```

### 方法3：在 IDE 中运行（IntelliJ IDEA / WebStorm）

1. **配置运行配置**
   - 点击 `Run` > `Edit Configurations...`
   - 点击 `+` 添加新配置
   - 选择 `Node.js`
   - 设置：
     - **Node interpreter**: 选择已安装的 Node.js
     - **Working directory**: 项目目录
     - **JavaScript file**: `server.js`
   - 点击 `OK`

2. **运行**
   - 点击运行按钮或按 `Shift + F10`

## 验证服务器是否运行

打开浏览器访问：`http://localhost:3000`

如果看到页面正常显示，说明服务器运行成功。

## 常见问题

### Q: 提示 "npm 不是内部或外部命令"
**A:** 需要先安装 Node.js
- 访问 https://nodejs.org/
- 下载并安装 LTS 版本
- 重启命令行后重试

### Q: 提示端口被占用
**A:** 修改 `server.js` 中的端口号
```javascript
const PORT = 3001; // 改为其他端口
```

### Q: 依赖安装失败
**A:** 尝试使用国内镜像
```bash
npm install --registry=https://registry.npmmirror.com
```

## 停止服务器

在运行服务器的命令行窗口中按 `Ctrl + C`
